CS140A11 - Compilation Process: From Source to Executable

Learning Objectives

  • Understand the 4 stages of C++ compilation: Preprocessor → Compiler → Assembler → Linker
  • Use g++ flags to produce each intermediate file
  • Examine the output of each stage (.i, .s, .o, a.out)

Compilation Stages

Step 1: Preprocess (-E)

g++ -E main.cpp -o main.i

Expands #include headers and macros. The output main.i is a large text file with all headers inlined.

Step 2: Compile to Assembly (-S)

g++ -S main.cpp -o main.s

Translates C++ into assembly language. The output main.s contains assembly directives like .text, .globl, and CPU instructions.

Step 3: Assemble to Object File (-c)

g++ -c main.cpp -o main.o

Converts assembly into machine code (binary). The output main.o is a binary file — not human-readable, but string literals like "Hello" can be found with the strings command.

Step 4: Link to Executable

g++ main.o -o a.out

Links the object file with system libraries to produce the final executable a.out.

Step 5: Run

./a.out > result.txt

Run the executable and save the output to result.txt.

What to Submit

You must commit and push the following files to your GitHub repository:

  • main.cpp — your source code (already provided)
  • main.i — preprocessed output
  • main.s — assembly output
  • main.o — object file
  • result.txt — program output
git add main.cpp main.i main.s main.o result.txt
git commit -m "completed all compilation stages"
git push
